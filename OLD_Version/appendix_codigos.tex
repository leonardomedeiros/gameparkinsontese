\chapter{Principais Algoritmos do Arcabouço} \label{app:codigos}

%Um maior detalhamento da estrutura do arcabouço pode ser visto no diagrama de classes resumido na Figura~\ref{img:classd}. As informações enviadas ao \textit{web service} são processadas pela classe \texttt{DataResource}, que são enviadas para o módulo \emph{Gerenciador de Dados} (classe \texttt{DataManager}) e são gravadas pelo \emph{Gerenciador do Banco de Dados} (classe \texttt{DatabaseManager}) no \emph{Banco de Dados} para análise posterior, ou enviadas diretamente para o \emph{Analisador de Dados} (classe \texttt{DataAnalyzer}) para análise. As classes \texttt{FilterModule}, \texttt{ParserModule} e \texttt{WriterModule} implementam, respectivamente, os módulos de filtragem, interpretação e escrita de dados. \texttt{MovementAnalyzer}, \texttt{TremorAnalyzer} e \texttt{RuleManager} são o analisador de movimentos, analisador de tremor e gerenciador de regras, respectivamente. \texttt{IWriter} é a interface que encapsula um arquivo de dados. \texttt{WriterImpl} reúne o comportamento comum das classes que encapsulam arquivos de dados \texttt{ARFFWriter}, \texttt{CSVWriter} e \texttt{DATWriter}. Todos os elementos são abordados nas seções a seguir.
